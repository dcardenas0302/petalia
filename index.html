Petalia

A programming language based on flowers.

More on GitHub Try It Yourself Language Reference Demo Programs Get Started

Petalia is a flower-themed educational programming language where every data type is a flower, output blooms as visual bouquets and gardens, and syntax is designed to be readable and beginner-friendly — because code should grow, not intimidate.

Built for CS420 by Diego Cardenas & Reinaldo Roldan — Team Petalia.

🌱Try it yourself

hello_world.petal

# Hello World in Petalia

tulip msg = "Hello, World!"

greeting.add("tulip", "Hello, World!")

🌸Language reference

Types

KeywordMeaningExample
roseInteger / Floatrose x = 42
tulipStringtulip name = "Alice"
daisyBooleandaisy flag = petal
lilyListlily nums = [1, 2, 3]
bouquetOutput collectionbouquet b = new bouquet()
gardenFull scene / functiongarden myFn(x) { ... }

Boolean & null literals

  • petal → true
  • wilted → false
  • bare → null / None

Output

bloom <expression>

Strings print as text. Bouquets and gardens render as visual flower displays.

Functions

garden functionName(tulip param1, rose param2) {
    ...
    return value
}

Control flow

# conditional
if (condition) { ... } else { ... }

# while loop
while (condition) { ... }

# for-in loop
for (item in list) { ... }

Built-in functions

  • len(list) — length of a list
  • str(x) — convert to string
  • int(x) — convert to integer
  • range(n) / range(start, end) — generate a number range
  • abs(x), max(...), min(...), sqrt(x)

lily (list) methods

  • .add(value) — append
  • .remove(value) — remove first occurrence
  • .pop(index) — remove and return at index
  • .len() — length
  • .contains(value) — membership check

bouquet methods

  • .add(flower_type, label) — add a named flower
  • .render() — get the string rendering
  • .size() — number of flowers

garden methods

  • .add(flower_type, label) — add a flower to the scene
  • .render() — get the visual rendering
  • .size() — number of items

🌷More examples

greet.petal — garden functions & bouquet returns

garden greet(tulip name) {
    bouquet b = new bouquet()
    b.add("rose",  "Hello, " + name + "!")
    b.add("lily",  name)
    b.add("daisy", "Welcome to the garden!")
    return b
}

bloom greet("Petalia")
bloom greet("Diego")
bloom greet("Reinaldo")
output — greet("Petalia")🌸 Bouquet: 🌹 [rose] Hello, Petalia! 🌺 [lily] Petalia 🌼 [daisy] Welcome to the garden!

fizzbuzz.petal — while loops, conditionals, garden output

# orchid=FizzBuzz  sunflower=Fizz  daffodil=Buzz  rose=number

rose i = 1
garden field = new garden()

while (i <= 15) {
    daisy by3 = (i % 3) == 0
    daisy by5 = (i % 5) == 0

    if (by3 and by5) {
        field.add("orchid",    str(i) + " → FizzBuzz")
    } else {
        if (by3) { field.add("sunflower", str(i) + " → Fizz") }
        else {
            if (by5) { field.add("daffodil", str(i) + " → Buzz") }
            else     { field.add("rose", str(i)) }
        }
    }
    i = i + 1
}
bloom field

fibonacci_garden.petal — functions, while loops, list indexing

garden fibonacci(rose n) {
    lily seq = new lily()
    rose a = 0
    rose b = 1
    rose count = 0
    while (count < n) {
        seq.add(a)
        rose temp = a + b
        a = b
        b = temp
        count = count + 1
    }
    return seq
}

lily fibs = fibonacci(10)
rose idx = 0
while (idx < 10) {
    bloom "F(" + str(idx) + ") = " + str(fibs[idx])
    idx = idx + 1
}
output🌷 F(0) = 0 🌷 F(1) = 1 🌷 F(2) = 1 🌷 F(3) = 2 🌷 F(4) = 3 🌷 F(5) = 5 🌷 F(6) = 8 🌷 F(7) = 13 🌷 F(8) = 21 🌷 F(9) = 34

🌿Getting started

Requirements

Python 3.7 or higher. No external packages needed.

1 — Clone the repository

git clone https://github.com/dcardenas0302/petalia/tree/website-branch && cd petalia

2 — Run a single program

python interpreter.py hello_world.petal

3 — Run all demos at once

python interpreter.py --demo
Tip: Start with hello_world.petal, then work your way up to fibonacci_garden.petal and grade_tracker.petal for the full garden experience.

🌻Language design notes

  • Lexical scoping with block-level variable visibility
  • Interpreted — runs top to bottom via a hand-written lexer, parser, and tree-walk evaluator in a single Python file
  • Supports Imperative (while / for), Functional (garden definitions, return values), and Declarative (bouquet / garden composition) paradigms
  • Visual output rendered using Unicode box-drawing characters and emoji
  • File extension: .petal
